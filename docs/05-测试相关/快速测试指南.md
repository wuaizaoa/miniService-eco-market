# 🚀 快速测试指南

## 📋 前置检查清单

在运行测试之前，请确保：

- [ ] MySQL 已启动，数据库 `lesson` 已创建
- [ ] Redis 已启动（默认端口 6379）
- [ ] Nacos 已启动（默认端口 8848）
- [ ] JDK 1.8+ 已安装
- [ ] Maven 3.x 已安装
- [ ] Python 3.7+ 已安装（用于API测试）

---

## 🛠️ 快速开始（3步）

### 第1步：编译项目
```bash
cd eco-market
mvn clean install -DskipTests
```

### 第2步：启动服务（4个终端窗口）

**终端1 - 启动用户服务 (8081)**
```bash
cd user-service/user-service-start
mvn spring-boot:run
```

**终端2 - 启动商品服务 (8082)**
```bash
cd product-service/product-service-start
mvn spring-boot:run
```

**终端3 - 启动订单服务 (8083)**
```bash
cd order-service/order-service-start
mvn spring-boot:run
```

**终端4 - 启动支付服务 (8084)**
```bash
cd payment-service/payment-service-start
mvn spring-boot:run
```

### 第3步：运行API测试
```bash
cd api-tests

# Windows
python -m venv venv
venv\Scripts\activate.bat
pip install -r requirements.txt
pytest -v
```

## 🎯 常用测试命令

### 运行单个服务测试
```bash
# 仅测试用户服务
pytest test_user_service.py -v

# 仅测试商品服务
pytest test_product_service.py -v

# 仅测试订单服务
pytest test_order_service.py -v

# 仅测试支付服务
pytest test_payment_service.py -v
```

### 运行端到端测试（完整购物流程）
```bash
pytest test_e2e_flow.py -v
```

### 生成HTML测试报告
```bash
pytest -v --html=reports/report.html --self-contained-html
```

### 运行性能测试
```bash
python performance_test.py
```

## 🔍 验证服务是否启动成功

### 检查服务状态

打开浏览器或使用 curl 访问：

| 服务 | 健康检查URL | 预期结果 |
|------|------------|---------|
| 用户服务 | http://localhost:8081/api/user/1 | JSON响应 |
| 商品服务 | http://localhost:8082/api/product/1 | JSON响应 |
| 订单服务 | http://localhost:8083/api/order/1 | JSON响应 |
| 支付服务 | http://localhost:8084/api/payment/1 | JSON响应 |

### 使用 curl 测试
```bash
# 测试用户服务
curl http://localhost:8081/api/user/1

# 测试商品服务
curl http://localhost:8082/api/product/1
```

## 📱 使用 Postman 手动测试

### 用户注册
- **URL**: `POST http://localhost:8081/api/user/register`
- **Body**:
```json
{
  "username": "testuser",
  "password": "password123",
  "email": "test@example.com",
  "phone": "13800138000"
}
```

### 用户登录
- **URL**: `POST http://localhost:8081/api/user/login`
- **Body**:
```json
{
  "username": "testuser",
  "password": "password123"

```

- **Body**:

## ⚠️ 常见问题

### 问题1：服务启动失败，提示找不到依赖
**解决方案**：确保先运行 `mvn clean install -DskipTests`

### 问题2：连接数据库失败
**解决方案**：检查 application.yml 中的数据库配置是否正确

### 问题3：Redis连接失败
**解决方案**：确保Redis已启动，检查Redis配置

### 问题4：Nacos连接失败
**解决方案**：确保Nacos已启动，检查Nacos地址配置
